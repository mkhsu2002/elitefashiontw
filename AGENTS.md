# 專案協作規則

一律用繁體中文回覆我。

所有專案通用規則：

1. 只要本次變更包含後端 schema、migration、資料表欄位、RPC、RLS、Edge Function、API 回傳格式或前後端資料契約，合併或發布時就絕對不可只更新前端或只更新部分環境。
2. 發布前必須逐一確認：
   - 程式碼所在分支已合併到正確目標分支
   - 正式環境 / dev 環境所需 migration 已實際套用
   - 正式環境 / dev 環境所需 functions / API 已實際部署
   - 前端目前使用的流程，與正式環境實際部署的後端版本一致
3. 若前端已切到新流程，但正式後端仍停在舊版，必須視為重大錯誤風險，優先排查並修復，不可假設只是顯示問題。
4. 涉及點數、帳本、付款、訂單、配額、佇列、狀態同步時，除了主資料表，還必須同步驗證：
   - 帳本 / log / audit 是否有正確寫入
   - 前端顯示是否即時刷新
   - 失敗、回補、重試、降級流程是否一致
5. 若本次事故已證明「環境版本不一致」曾造成問題，後續同類專案必須主動做 deployed code / deployed schema / live behavior 的交叉驗證，不可只看本地 repo。
6. 撰寫前端的任何文字內容時，都要避免將內部思考過程及文案策略構思相關的內部文字透露在前端。
7. 任務告一個段落、完成可交付變更後，必須主動 commit 並推上 GitHub `main`，不得等使用者再次提醒。
8. 特別指令專案生成的前導頁、品牌頁、產品頁或結構明顯不同的一頁式內容，不可納入一般文章封面重複圖片的 AI 生圖置換批次；目前明確排除 `outdoor-escapes/horizon-x-space-travel-neck-pillow.html`。除非使用者明確指定，不可替換這類頁面的文中展示圖。
9. 全站人物圖片預設以歐美白人與亞洲華人為主；除非文章主題明確需要，不可出現中東、印度、南亞人物造型，也避免 hijab、頭巾、turban、sari、abaya、伊斯蘭或南亞宗教/民族風格裝飾。
10. 後續批次生成或置換封面時，要刻意拉開視覺風格與色調，不可大量重複同一種室內暖光、商務沙發、筆電桌面構圖。需穿插更豐富的自然色與場景，例如陽光海洋、森林原野、花園花草、城市街景、山景、季節色彩與更鮮明的戶外生活感，讓全站封面呈現五花八門的豐富感。
