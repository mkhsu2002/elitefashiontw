# 專案協作規則

本檔只保留 Elite Fashion TW 專案特有規則；通用發布一致性、前端文字防洩漏、skills/rules 治理請遵守全域 `/Users/mkhsu/.codex/AGENTS.md`。不要自動 commit / push；只有使用者明確要求發布或推送時才執行 git 上傳流程。

1. 特別指令專案生成的前導頁、品牌頁、產品頁或結構明顯不同的一頁式內容，不可納入一般文章封面重複圖片的 AI 生圖置換批次；目前明確排除 `outdoor-escapes/horizon-x-space-travel-neck-pillow.html`。除非使用者明確指定，不可替換這類頁面的文中展示圖。
2. 全站人物圖片預設以歐美白人與亞洲華人為主；除非文章主題明確需要，不可出現中東、印度、南亞人物造型，也避免 hijab、頭巾、turban、sari、abaya、伊斯蘭或南亞宗教/民族風格裝飾。
3. 後續批次生成或置換封面時，要刻意拉開視覺風格與色調，不可大量重複同一種室內暖光、商務沙發、筆電桌面構圖。需穿插更豐富的自然色與場景，例如陽光海洋、森林原野、花園花草、城市街景、山景、季節色彩與更鮮明的戶外生活感，讓全站封面呈現五花八門的豐富感。
4. 撰寫 momo 聯盟行銷文章前，必須先查 `automation/momo-brand-recommendation-tracker.csv` 與 `automation/momo-affiliate-content-architecture.md`，確認品牌是否已被推薦、近期是否重複曝光、應屬於主角品牌、配角品牌、背景品牌或暫緩品牌。
5. momo 聯盟文章不可只重複推薦最高價值品牌；每個小主題系列需刻意覆蓋不同品牌。A+ / A 品牌可作主推，B 品牌可作配角與比較維度，C 品牌需查證後才可弱露出，D 品牌不得進正式文章。前台文字不可出現「主角品牌、配角品牌、SEO、品牌池、矩陣、覆蓋率、批次」等內部規劃語言。
6. momo 聯盟文章須歸入既有大分類，不新增大分類；戶外移動歸 `outdoor-escapes`，通勤穿搭與鞋包歸 `casual-chic`，睡眠護具與恢復歸 `wellness-movement`，香氛照明收納咖啡茶與送禮歸 `lifestyle-culture`，拍攝設備、外接螢幕、筆電與 AI 工作角落歸 `ai-innovation`。若跨分類，以讀者主要搜尋意圖決定。
7. momo 聯盟文章完成後，必須回填品牌推薦追蹤表的文章 slug、live URL、提及次數與最後提及日期；文章若含導購連結，需在正文最末端固定放置可見導購揭露，連結使用 `rel="sponsored nofollow"`。涉及健康、護具、保健、食品、睡眠、長照、美容成分時，不可承諾療效、減重、醫療、保健或抗老效果。
8. momo 聯盟文章正式撰寫不得使用 `gpt-5-mini` 作為預設 writer model；預設 writer 使用 `gpt-5.4`，planner 可使用 `gpt-5.2` 以平衡內容表現與成本。若因成本或限額降級，需明確記錄並人工複核文章品質。
9. 每篇 momo 聯盟文章完成後，需生成同一張圖像內容作為文章封面與 OG/Twitter 預覽圖，建議尺寸 1200 x 630。Codex 人工產文時應調用 Codex 可用的圖片生成能力；GitHub Actions 自動排程若尚未接入圖片 API，需標記 `needs-cover-image`，不可假裝已有新封面。
10. Elite Fashion 文章以「編輯團隊」名義發布。內容生成需先有雜誌式引言與讀者問題框架，再進入商品、品牌或工具；不得一開頭就自言自語式羅列產品使用情境，也不得在前台暴露 AI 身分、prompt、SEO 策略、分潤策略或內部流程。文章需視為經過知識研究、文學美化、發布前編審、主編風格確認後才發布。
11. GitHub Actions 的 `content-scheduler.yml` / `content-command.yml` 產文流程會在正式上線與 Google Sheets 紀錄後呼叫電子報寄送；若由 Codex 手動寫文並直接推上 `main`，不會因 `git push` 自動寄給訂閱會員，需另行執行電子報寄送流程或明確記錄本篇未寄送。
12. 後續新文章的內部目標受眾調整為 30 到 50 歲白領菁英與高效率生活讀者；但前台標題、導言、H2、FAQ 與 CTA 不可直白寫出「30+、40+、45+、50+、熟齡、銀髮、老人」等會框定讀者的字眼，除非主題本身就是年齡、醫療、法規或人生階段議題。一般文章需以通勤、會議、週末、旅行、居家、工作角落、恢復、送禮等開放情境表述，讓不同年齡讀者都能自然進入。
13. 新增、改寫或批次生成文章時，必須遵守 `automation/seo-content-execution-rules.md`：主體內容、FAQ、CTA、導購揭露與重要內鏈需存在於初始 HTML；不得使用 `nosnippet`、`data-nosnippet` 或會阻擋搜尋摘要的設定；不可把正文、導購連結或商品資訊只放在 JavaScript 動態載入內容中。
14. 每篇新文章都必須有「只有 Elite Fashion 編輯團隊能寫得出來」的具體判斷段落，例如選購順序、常見錯誤、台灣情境、使用頻率或取捨比較；若文章只是通用 AI 式整理，需退回重寫，不得發布。
15. 內容圖片需撰寫具描述性的 alt text；若新增影片或影片頁，需提供可見摘要或逐字稿重點，且只有在頁面確實含影片並有可見內容支撐時，才可加入 `VideoObject` schema。本站目前不是實體店家或在地服務業，不得為 SEO 虛構 Google Business Profile、地址、電話或營業時間。
16. 後續新聯盟文章的前台標題可加品牌前綴時，一律使用 `Elite Fashion｜`，不要使用 `Elite Fashion TW｜`。已發布測試文章不回頭修改，作為版型檢視紀錄保留。
17. 後續新聯盟文章的文中與文末 CTA 需明顯可點擊：按鈕文字使用較高字重，可搭配箭頭或小圖示，且每篇文章的 CTA 呈現可在色彩、圖示或排列上稍微變化；但不得使用誇張促銷語、倒數、折扣保證或造成壓迫感的文案。
