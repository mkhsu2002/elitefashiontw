# Elite Fashion 選題機制

## 核心原則
- 先止血，後增長。
- queue 優先，fallback 次之。
- fallback 不是自由發揮，而是「避開舊傷口之後」才允許擴張。
- 不追單一工具熱詞，不用外部案例名稱當預設主題。
- 選題必須同時滿足「這個站的讀者會想看」與「這個站本來就接得住」。

## queue 為空時的實際流程
1. 先讀 `automation/life-proposals-pruning-map.md`
- 凡是已被標成 `降權`、`刪除` 或待 `合併` 的 legacy 意圖，不可再用相似角度重寫新文。
- 只有當某一題是明確的 owner page 重建題，才可以重回 backlog。

2. 再讀 `automation/topic-backlog.json`
- 只從 `fallbackTopics` 中挑選。
- 題庫本身已經過春夏策略篩選，不再讓系統自己從空白狀態亂想題目。
- `springSummer2026IssueBank` 是人工策展候選庫，用來排 queue 或擴充 fallback；排程不應一次直接吃完整候選庫。

3. 先做排除，再做加分
- 排除：
- 與既有標題高度重複。
- 命中 `主權協議 / 權威 / 革命 / 只換年份` 等舊公式。
- 命中被暫時降權的題型，例如秋冬厚重材質、抽象勵志、輕醫療化題。
- 加分：
- 近期分類缺口。
- 春夏季節適配度。
- 商業與內連價值。
- evergreen 程度。

4. 最後做節奏輪替
- fallback 每週 3 篇的理想節奏：
- 1 篇人生重整 / 關係決策
- 1 篇健康恢復、穿搭或戶外的實用解法
- 1 篇 AI、趨勢或設計翻譯題
- 若最近已連續出現同類關鍵字，例如連續多篇都在談離婚、AI 或睡眠，就透過 `relatedKeywords` 扣分，避免短時間內重複轟炸同一意圖。

## 目前新版 backlog 的運作邏輯

### 1. 季節權重
- `spring-summer` 題型有最高加權。
- `all-season` 次之。
- `fall-winter` 題型在春夏檔期自動降權。

### 2. 類別配額
- `lifestyle-culture`：27%
- `wellness-movement`：20%
- `casual-chic`：18%
- `outdoor-escapes`：14%
- `ai-innovation`：14%
- `runway-trends`：5%
- `designer-perspective`：2%

### 3. 舊意圖黑名單
- `suppressedLegacyFamilies` 用來記錄不該再擴張的舊題型：
- life-proposals 公式標題群
- 泛財富自由 / 被動收入勵志題
- 年齡焦慮式外貌單點優化題
- 高風險輕醫療化題

### 4. 人工指定例外
- `品牌專案`
- `法規、醫療與投資高風險題`
- `外部合作指定 CTA 題`
- 以上題型不應由 fallback 自動亂出，應由人工下 queue 或直接生成指令處理。

## 這套機制要解決的，不只是「寫什麼」
- 它同時要回答：
- 現在什麼不該再寫？
- 哪些 legacy 題型應該停損？
- 哪些新題目雖然熱門，但和本站品牌不一致？
- 哪些主題應該用 owner page 重建，而不是再新增一篇薄頁？

## 春夏檔期最優先的 fallback 題型
- 春夏離婚後的生活重整與社交節奏
- 白領工作者的 AI 採用與工作重整
- 職涯副業與第二曲線的低風險測試
- 春夏熱感、浮腫與睡眠恢復
- 肌力、步行耐力與旅行體力
- 春夏旅行膠囊衣櫥與移動穿搭
- 鞋包重量、久走舒適度與通勤節奏
- 城市旅行、台灣輕戶外與週末修復
- 居家閱讀、工作角落與生活秩序重整
- AI 對客服、行政與內容工作的低壓導入
- 把秀場趨勢翻譯成日常衣櫥
- 設計師觀點下的個人美學與辨識度

## 分類名稱更新
- `wellness-movement` 前台名稱已改為「健康恢復」。
- 相關題目應使用睡眠、熱感、肌力、步行、旅行疲勞與身體節奏語境，不再把分類窄化成單一運動類型，也不主動用年齡標籤框定讀者。
